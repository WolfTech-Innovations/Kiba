## 2025-05-14 - [CI Workflow Optimization: Removing Unused Host-side Dependencies]
**Learning:** GitHub Actions workflows often include "just in case" dependencies (like build-essential, meson, and Gtk dev headers) that add significant overhead to the environment setup. In KibaOS, since the actual ISO build happens inside a chroot/Docker environment with its own package list, these host-side dependencies were redundant. Removing them saves download time, disk space, and minimizes the risk of host-side version conflicts.
**Action:** Always audit the `apt-get install` steps in CI workflows. If a build process is containerized or uses `live-build` chroot hooks, the host runner only needs orchestration tools (like `git`, `curl`, `xorriso`), not the full compiler toolchain or GUI libraries.
